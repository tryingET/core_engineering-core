# Local release draft — engineering-core v0.3.0

## Title

`engineering-core v0.3.0 — 2,000 ft discipline completion`

## Release body

This release completes the 2,000 ft cross-language engineering guidance layer by adding the remaining shared disciplines for services, AI/ML, performance, releases, data, domain modeling, and design-pattern vocabulary.

### Breaking Changes

- None. This is an additive pre-1.0 guidance and catalog release.

### Added

- `service-api` discipline for service/API boundaries, contracts, auth, errors, idempotency, jobs, health/readiness, migrations, deployment, and rollback.
- `ai-ml` discipline for model assets, inference/runtime boundaries, evals, dataset/prompt/model provenance, fallback, privacy, safety claims, reproducibility, and evidence.
- `performance` discipline for profiling, budgets, benchmark hygiene, regression gates, frontend/backend/GPU measurements, and evidence.
- `release-package` discipline for semantic versions, changelogs, artifact provenance, publishing, compatibility, migration releases, deprecation, and rollback.
- `data-governance` discipline for data authority, schemas, lineage, lifecycle, quality, privacy, migrations, projections, controlled-data references, and semantic-owner boundaries.
- `domain-modeling` discipline for domain vocabulary, invariants, workflows, state transitions, aggregate boundaries, anti-corruption layers, controlled-vocabulary boundaries, and pattern placement.
- `design-patterns` discipline as a 63-entry cross-language pattern vocabulary index.
- CLI/catalog/profile coverage for all new disciplines.
- Tests for new discipline availability, catalog sync, architecture wikilinks, and the 63 design-pattern entries.

### Changed

- Updated README discipline lists, typical combinations, and CLI examples.
- Added concise lane pointers for when to load the new disciplines.
- Clarified that React/Svelte/Vue/etc. framework addenda remain deferred until recurring repo evidence justifies them.
- Added front matter to the Rust build-graph addendum so docs strict checks pass.

### Fixed

- Tightened wording around ROCS/controlled vocabulary boundaries using wikilinks instead of copied vocabulary definitions.
- Avoided machine-absolute architecture links by using `~/ai-society/...` wikilinks.

### Agent migration notes

- Existing consumers do not need a breaking migration. Consumers that select disciplines from `catalog.json` may opt into the new discipline IDs as needed.
- Use `data-governance` and `domain-modeling` for data/domain concerns instead of overloading `local-first-data` or `specification-and-dsls`.
- Use `design-patterns` as a vocabulary index only; invariant-specific guidance still belongs in the owning discipline or lane.
