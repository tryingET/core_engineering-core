# Local release draft — engineering-core v0.2.0

## Title

`engineering-core v0.2.0 — hard rename and discipline adoption surface`

## Release body

This release completes the hard rename into the engineering-core naming family and adds the first adoption surface for lane plus discipline guidance.

### Breaking Changes

- The previous package, CLI, import, lane-file, local-override, and policy names are removed rather than aliased. Consumers must migrate to the engineering-core names before relying on this release.

### Added

- Cross-language discipline docs and CLI commands for listing, showing, and resolving discipline paths.
- Adoption artifacts: `docs/adoption.md`, `catalog.json`, and `templates/engineering.local.template.md`.
- CLI tests for lane, discipline, path, and no-alias behavior.

### Changed

- Version bumped to `0.2.0` for the pre-1.0 breaking rename.
- Lane docs and symlink surfaces now use engineering-core naming.

### Fixed

- Removed stale legacy package and command entry points from the active package surface.

### Agent migration notes

- Resolve the breaking rename with [v0.2.0 migration map](migrations/v0.2.0.md) before updating prompts, templates, fixtures, consumers, or operator scripts to this release.
